## User Spec: Сведения об образовательной организации

## Что делаем

Создать страницу «Сведения об образовательной организации» в соответствии с требованиями ФЗ-273 для организаций с лицензией ДПО.

**Страница:**
- Файл: `svedeniya-ob-obrazovatelnoy-organizatsii.html`
- Тип: статический HTML с заглушками контента
- Дизайн: по дизайн-коду «Фрегат»

## Зачем

### Юридическое требование
По ФЗ-273 организации с лицензией ДПО обязаны публиковать на официальном сайте:
- Сведения о лицензии (номер, дата выдачи, срок действия)
- Полное и сокращённое наименование
- Юридический и фактический адрес
- Реквизиты (ИНН, ОГРН, КПП)
- Уровень образования
- Образовательные программы с трудоёмкостью
- Преподавательский состав с квалификацией
- Материально-техническое оснащение

### Доверие клиентов
Потенциальные студенты проверяют легитимность школы перед покупкой курса. Страница с официальными сведениями повышает доверие.

## Acceptance Criteria

### Функциональные требования
1. Страница создана по пути `svedeniya-ob-obrazovatelnoy-organizatsii.html`
2. Использована единая навигация (header + footer) как на главной странице
3. В header добавлена иконка поиска (заглушка — клик без действия)
4. Хлебные крошки: `Главная → Сведения об образовательной организации`
5. Все секции из дизайн-кода используются (цвета, компоненты, утилиты)

### Контент (с заглушками)
1. **Заголовок H1**: «Сведения об образовательной организации»
2. **Об организации**: ООО «Фрегат», история, миссия (placeholder)
3. **Лицензия**: Л035‑01217‑26/03829175 от 25.11.2025 (placeholder скана)
4. **Программы**: «Тендеры с нуля» (placeholder описания)
5. **Преподаватели**: 2-3 placeholder-профиля
6. **Материально-техническая база**: описание платформы (placeholder)
7. **Контакты**: телефон, email, адрес (placeholder)
8. **Реквизиты**: ИНН, ОГРН, КПП, юридический адрес (placeholder)

### Технические требования
1. Микроразметка schema.org (CollegeOrUniversity)
2. OG теги для соцсетей
3. Адаптивность (mobile-first)
4. Валидный HTML5

### Интеграция
1. Ссылка на страницу добавлена в footer
2. Единый стиль с главной страницей

## User Flows

### Flow 1: Проверка лицензии (потенциальный студент)
1. Пользователь заходит на главную страницу
2. Скроллит до footer, видит ссылку «Сведения об образовательной организации»
3. Кликает, попадает на страницу
4. Видит лицензию, реквизиты, программы
5. Возвращается на главную для покупки курса

### Flow 2: Проверка organs (контролирующий орган)
1. Ревизор открывает сайт
2. Ищет раздел с официальными сведениями
3. Находит в footer или через поиск (когда будет реализован)
4. Проверяет наличие обязательных разделов

## Ограничения

### Что НЕ делаем
- Поиск по сайту (только заглушка иконки)
- Реальные документы (только placeholder для скана лицензии)
- Динамический контент (всё статично)
- Админку для управления контентом

### Что делаем с заглушками
- Все тексты — placeholder (ООО «Фрегат», ИНН, адрес)
- Фото преподавателей — placeholder
- Скан лицензии — placeholder

## Риски

| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Отсутствие реальных данных | Высокая | Низкое | Placeholder позволяет быстро запустить |
| Изменения в ФЗ-273 | Низкая | Среднее | Структура page учитывает текущие требования |
